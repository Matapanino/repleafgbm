## Analysis and conclusions (2026-06-11, fair v2)

This revision gives the LightGBM base early stopping on the same validation
set the native models use, and lets the replay stage early-stop as well —
removing the capacity-asymmetry caveat of the Phase 3 run.

**1. Phase 3's core finding survives fair tuning.** On both regression
datasets, refitting LightGBM's own early-stopped routes with embedded-linear
leaves improves on LightGBM itself:

- piecewise_linear: 0.4864 → 0.4304 (plr; −11.5%) / 0.4516 (identity)
- friedman1: 1.1921 → 1.1689 (identity; −1.9%)

The within-router comparison is now clean on both sides: the
representation-conditioned-leaf contribution is real and not an artifact of
an untuned base.

**2. The native router wins on regression even after fair tuning**
(0.4074 / 1.1282), and with far fewer trees (136 / 79 vs 210-276 replayed).
Plausible drivers, untested: the native configuration grows smaller trees
(num_leaves=16, lr=0.1) whose leaves keep enough samples for linear fits,
while LightGBM's 31-leaf trees fragment the data; and native trees are
grown *interleaved* with embedded-leaf updates, so later trees compensate
the leaves' residuals. Disentangling these is a research question, not a
bug; on synthetic data this ranking should not be over-read.

**3. Regression sanity still holds; binary "sanity" is approximate by
design.** routerx constant ≈ lightgbm (0.4856 vs 0.4864; 1.1906 vs 1.1921)
with the small gap from l2_leaf=1.0 — and on binary the replayed constants
(0.2823) actually edge out LightGBM (0.2850): the ridge damping acts as
mild extra regularization on logistic Newton steps. Exact equality is only
claimed (and tested) for regression with l2≈0.

**4. Binary classification: embedded leaves do not help on this dataset.**
All six models sit within 0.005 logloss; native *constant* is best
(0.2804) and both embedded variants land mid-pack. The binary logit here is
step + linear — structure trees already capture — and the h-weighted ridge
shrinks leaf-linear gains when p(1−p) is small. A binary dataset with
strong within-region smooth structure is needed before concluding anything;
flagged as follow-up.

**5. Replay early stopping works as intended**: embedded replays stop at
96-276 routes depending on dataset (vs consuming everything in Phase 3),
and prediction uses the best iteration.

**Caveats.** Synthetic data, two seeds, one native hyperparameter setting;
the native-vs-extracted ranking is configuration-sensitive. The robust,
repeated finding is the within-router one: embedded leaves ≥ constant
leaves on the same routes for regression.
