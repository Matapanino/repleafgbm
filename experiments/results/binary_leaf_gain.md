## Analysis and conclusions (2026-06-12)

**Hypothesis verdict: partially confirmed mechanically, rejected as the
actionable cause.**

1. **The ESS collapse is real but modest.** Binary per-leaf effective sample
   size decays to ~64% of the regression value by rounds 76-100 (321 vs 500)
   as confident regions drive h = p(1-p) toward 0 — the mechanism exists.
   But linear-leaf fractions decay almost identically for both tasks
   (0.99→0.43 vs 0.95→0.47), so h-weighting is not what disables the
   linear fits.
2. **No reweighting remedy buys anything.** Across both datasets every
   remedy lands within ~0.004 logloss of the default and none beats
   constant leaves: l2 ∈ {1.0, 0.25, 0.1} are statistically
   indistinguishable (so the "shared l2 over-regularizes logistic fits"
   sub-hypothesis also fails); the raised Hessian floor is mildly *harmful*
   (it distorts targets t = -g/h in exactly the confident regions);
   damped weighting (α=0.5, α=0) is a wash.
3. **The telling diagnostic is the weight-norm trajectory.** For regression,
   mean ||w|| collapses 0.75 → 0.09 as residuals shrink — leaves stop
   finding linear signal when there is none left. For binary it *stays* at
   ~0.45 through round 100: late leaves keep fitting sizable linear models
   whose out-of-sample value is nil (the guard and ridge contain the damage;
   early stopping cuts the run). Interpretation: after the router carves
   probability regions, the within-leaf Newton targets in a binary task
   carry little *recoverable* smooth structure — what remains is mostly
   noise with the right scale to attract a fit.
4. **Decision: no default change.** Every candidate is within seed noise of
   the current defaults, and our policy is evidence-based defaults only.
   The practical guidance (now documented): for binary tasks
   `leaf_model="constant"` is an equally accurate and cheaper choice;
   embedded leaves remain the default for API consistency at a measured
   cost of ≤0.4% logloss on every binary dataset tested (adult here:
   constant 0.2909 vs default embedded 0.2920).
5. **adult re-evaluation**: consistent with Phase 8b
   (real_data_validation.md) — the residual gap to LightGBM-native-cat on
   adult is not addressable by leaf-fit reweighting or l2; remaining
   levers are categorical-pipeline refinements and router capacity.

**Where binary gains might still come from** (recorded, not started):
representations that linearize the *logit* within regions (learned
encoders), or vector/probability-calibrated leaf outputs — both are
existing roadmap tracks; this experiment removes "fix the h-weighting"
from the candidate list.

**Caveats.** Two seeds, one synthetic family + adult, single num_leaves
setting. The null result is consistent across all 16 config-dataset cells,
which is what gives it weight.
